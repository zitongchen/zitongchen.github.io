Java-IO学习笔记

发表于 2017-08-19 | 分类于 Java | 阅读次数

Java中的IO具有很多的类，在之前的学习中经常被其弄得不知所云，因此问度娘找了一幅JavaIO流的关系图，以加深对JavaIO的理解

File类

java.io.File类只用于表示文件（目录）的信息（名称，大小等），不能用于文件内容的访问

File读取一个目录，File有多个构造函数，不但可读取目录，同时还可以读取文件

RandomAccessFile

RandomAccessFile提供了对文件内容的访问，即可读文件，也可写文件，同时支持随机访问文件，可以访问文件的任意位置

Java文件模型

  • 在硬盘上的文件是以byte字节数组存储的，是数据集合
  • 打开文件有两种模式，一种是“rw”(读写),另一种是“r”(只读)

上述的RandomAccessFile使用了读写的方式，来进行对file的操作，打开文件时指针在开头pointer = 0

  • raf.write(int) —-> 只写一个字节（后8位），同时指针指向下一个位置，准备继续写入
  • int b = raf.read() —-> 读取一个字节

基本类型的序列化

  • 将类型int转换成4byte或将其他数据类型转换成byte的过程称为序列化，数据 —-> n byte
  • 将n 个 byte转换成一个数据的过程称为反序列化，n byte —–> 数据
  • RandomAccessFile提供基本类型的读写方法，可以将基本类型数据序列化到文件或者将文件内容反序列化为数据

Java的操作符：

  • &（与） 输入数据的对应位是1时，输出1，其他输出0
  • | （或）输入数据的对应位是0时，输出0，其他输出1
  • ^ （异或）数据数据的对应位相反时（即一个是0 ，一个是1），输出1，其他输出0
  • ~ （非）取反，输入1，输出0；输入0 ，输出1
  • << (左移位) 按照右侧指定的位数，向左移，低位补0
  • >> (带符号右移）操作数为正，高位补0，操作数为负高位补1
  • >>> (无符合右移）无论正负，高位补0

字节流

  • InputStream抽象了应用程序读取数据的方式
  • OutputStream抽象了应用程序写出数据的方式
  • EOF = End 读到-1就表示读到文本内容结尾

输入流基本方法：

  • int b = in.read();读取一个字节无符合填充到int底八位，-1是EOF
  • in.read(byte[] buf);从流中读取buf.length个字节数据放进数组中，返回读取到的字节总数
  • in.read(byte[] buf,int start, int size)

输出流基本方法：

  • out.write(int b) 写出一个byte到流，b到底8位
  • out.write(byte[] buf)就buf字节数据写入到流中
  • out.write(byte[] buf, int start, int size)

FileInputStream 具体实现了在文件上读取数据，读取的是byte数据，是对InputStream的第一层实现，实现了InputStream的基本方法

FileOutputStream 实现了向文件中写出byte数据的方法，是对OutputStream的第一层实现，实现了OutputStream的方法

在日常的使用中，文件的读取与写入都是使用字节数组批量处理字节

DataOutputStream/DataInputStream是对流功能的扩展，可以更加方便读写int,long,字符等数据类型; DataInputStream是继承了FileInputStream类,其构造函数需要一个输入流;DataOutputStream是继承了FileOutputStream类，其构造函数需要一个输出流

BufferedInputStream/BufferedInputStream 这两个流类为IO提供了带缓冲区的操作，一般打开文件进行文件的写入或读取操作时，都会加上缓冲，这种流模式提高了IO的性能

FileOutputStream —write()方法相当于一滴一滴地把水“转移”过去

DateOutputStream –>writeXxx()方法会方便一些，相当于一瓢一瓢的把水“转移”过去

BufferedOutputStream —>write()方法更方便，相当于一瓢一瓢的放到桶里，再一桶一桶的倒入另一个缸中，性能提高了

#字符流
java的文本（char）是16位无符号整数，是字符的unicode编码（双字节编码），文件是byte数组的数据序列，文本文件是文本（char）序列按照某种编码方案（utf-8,utf-16be,gbk）序列化为byte存储的结果

字符流（Reader Writer）： 操作的是文本文件，字符的处理，一次处理一个字符，字符底层仍然是字节序列

字符流的实现：

  • InputStreamReader 完成byte流解析为char流，按照编码解析；OutputStreamWriter 提供char流到byte流，按照编码处理
  • FileReader/FileWriter
  • 字符流的过滤器：BufferedReader —readLine 一次读一行；BufferedReader /PrintWriter —>一次写一行

#对象的序列化，反序列化

  • 对象序列化，就是将Object转换成byte序列，反之叫对象的反序列化

  • 序列化流(ObjectOutputStream),是过滤流—-writeObject;反序列化流(ObjectInputStream)—readObject

  • 序列化接口(Serializable)对象必须实现序列化接口 ，才能进行序列化，否则将出现异常;这个接口，没有任何方法，只是一个标准

  • transient关键字 :被transient修饰的成员变量不会自动被序列化，若想序列化该成员变量，需要手动序列化
