MongoDB学习之备份,恢复,监控

发表于 2017-03-21 | 分类于 MongoDB | 阅读次数

备份

备份的语法

  • -h:数据库服务器的地址，可以指定端口号

  • -d要备份的数据库

  • -o备份数据的存放位置

    可选参数：

    • mongodump –host xxxx –port xxxx 备份所以数据
    • mongodump –collectoin xxxx –db xxxx 备份指定的数据库集合

若执行mongodump的时候没有指定参数，会备份所以数据到bin下的dump\test

恢复

  • -h:port ：数据服务器地址
  • -d :需要恢复的数据库
  • -drop ：恢复数据库的时候先删除当前的信息
  • path :设置备份数据的位置
  • -dir: 指定备份数据的位置，但不可与path同时使用

若执行mongorestort的时候没有指定参数，会从bin下的dump\test恢复所以数据

监控

mongostat命令，监控增删改查耗时的执行情况

mongotop命令，监控数据库读写的时间 mongotop 10表示每隔10秒监控一次
