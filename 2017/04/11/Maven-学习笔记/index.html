Maven 学习笔记

发表于 2017-04-11 | | 阅读次数

Maven是一个项目管理与整合工具。Maven为开发者提供了一套完整的构建生命框架，开发团队可以快速的自动完成工程的基础构建配置，应为Maven使用了一个标准的目录结构和一个默认的构建声明周期。

总的来说,Maven简化了工程的构建过程，并对其标准化。其衔接了编译，发布，文档生成，团队合作和其他的任务

Maven的主要目的是为开发者提供

  • 一个可复用，可维护，更易理解的工程综合模型
  • 与这个模型交互的插件或者工具

Maven工程结构和内容被定义在一个XML文件中-pom.xml,是Project Object Model的简称，该文件是整个Maven系统的基础组件。

Maven是约定优于配置的，当创建Maven工程时，Maven会创建默认的工程结构，开发者只需把文件放到适当的工程结构便可。

以在MyEclipse 构建的一个Maven Javaweb为例说明Maven的工程结构

结构目录 配置文件
src/main/java Java源文件
src/main/resources 项目资源文件
src/test/java 测试源文件
src/test/resources 测试资源文件
target 编译文件等（自动生成）
src/main/webapp Webapp文件
…… …..

安装Maven

  • Maven是一个基于Java的工具，在使用Maven构建工程之前要配置Java的运行环境（关于如何安装Java运行环境自行百度）
  • 下载Maven文件 Maven 3.5.0 文件
  • 解压Maven 文件，把解压文件放置在你想安装Maven的目录
  • 设置Maven的环境变量，M2_HOME={Maven的安装目录},把 %M2_HOME%\bin添加到系统path变量后面
  • 在cmd命令窗口输入mvn --version（window系统）验证是否安装成功，若有Maven的版本信息输出证明安装成功
