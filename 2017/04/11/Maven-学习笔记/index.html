Maven 学习笔记

发表于 2017-04-11 | 阅读次数

Maven是一个项目管理与整合工具。Maven为开发者提供了一套完整的构建生命框架，开发团队可以快速的自动完成工程的基础构建配置，应为Maven使用了一个标准的目录结构和一个默认的构建声明周期。

总的来说,Maven简化了工程的构建过程，并对其标准化。其衔接了编译，发布，文档生成，团队合作和其他的任务

Maven的主要目的是为开发者提供

  • 一个可复用，可维护，更易理解的工程综合模型
  • 与这个模型交互的插件或者工具

Maven工程结构和内容被定义在一个XML文件中-pom.xml,是Project Object Model的简称，该文件是整个Maven系统的基础组件。

Maven是约定优于配置的，当创建Maven工程时，Maven会创建默认的工程结构，开发者只需把文件放到适当的工程结构便可。

以在MyEclipse 构建的一个Maven Javaweb为例说明Maven的工程结构

结构目录 配置文件
src/main/java Java源文件
src/main/resources 项目资源文件
src/test/java 测试源文件
src/test/resources 测试资源文件
target 编译文件等（自动生成）
src/main/webapp Webapp文件
…… …..

安装Maven

  • Maven是一个基于Java的工具，在使用Maven构建工程之前要配置Java的运行环境（关于如何安装Java运行环境自行百度）
  • 下载Maven文件 Maven 3.5.0 文件
  • 解压Maven 文件，把解压文件放置在你想安装Maven的目录
  • 设置Maven的环境变量，M2_HOME={Maven的安装目录},把 %M2_HOME%\bin添加到系统path变量后面
  • 在cmd命令窗口输入mvn --version（window系统）验证是否安装成功，若有Maven的版本信息输出证明安装成功

Maven Pom

Pom代表工程对象模型，它是使用Maven工作时的基本组建，是一个xml格式文件。存放在工程的根目录下，并命名为pom.xml。POM中包含关于工程和配置信息，Maven使用这些信息构建工程

Pom举例

每个工程只有一个POM文件

  • 每个POM文件需要project元素和三个必须字段：groupId,artifactId,version
  • 仓库的标识为groupId+artifactId+version

构建的生命周期

Maven构建工程会按照定义的阶段序列顺序执行每个阶段的注册目标，Maven有三个标准的生命周期：

  • clean
  • default(or build)
  • site

目标表示一个特定的，对构建和管理工程有帮助的任务，它可以有0个或多个构建阶段

例如：mvn clean packageMaven指挥按顺序执行clean -> package的构建阶段，而package以后的阶段maven不会执行

clean的声明周期

  • pre-clean
  • clean
  • post-clean

Default(or Build)主要生命周期

生命周期 描述
validate 检查工程配置是否正确
initialize 初始化构建状态，例如设置属性
complie 编译工程源码
test-complie 编译测试源代码到测试目录
test 使用适当的单元测试框架运行测试
package 获取编译后的代码，按照可发布的格式进行打包（war,jar）
verify 运行检查操作来验证工程包是有效的，并满足要求
install 安装工程包到本地仓库，该仓库可以作为本地其他项目的依赖
deploy 拷贝最终的工程包到远程仓库中，共享给其他工程人员

site 生命周期

Maven site 插件一般用来创建新的报告文档，部署站点等

site的生命阶段：

  • pre-site
  • site
  • post-site
  • site-deploy

Maven的配置文件

配置文件是一组配置集合，是用来覆盖Maven构建的默认配置。使用配置文件可以为不同阶段构建过程，例如生产环境和部署环境等

Profile主要有三种类型

类型 定义文件的位置
Per Pro 定义在每个工程的pom.xml中
Per User 定义在Maven设置的xml中(%user_home%/.m2/settings.xml
Global 定义在Maven全局配置文件中(%M2_HOME%/conf/settings.xml)

Profile在pom.xml中使用activeProfiles/profiles元素指定

Profile 的激活主要有，显式的使用命令控制台输入，通过maven设置

    Maven仓库有三种类型：

    修改本地仓库默认位置可以修改Maven的配置文件，在%M2_HOME%\conf目录中的settings.xml
